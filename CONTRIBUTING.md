# Contributing to Blink

## Setup

```bash
bundle install
```

Run the repository's Ruby test and command-validation workflow before finishing changes.

## Expectations

- Keep manifest behavior, CLI output, and history semantics stable and documented.
- Update `README.md` when the command surface or manifest contract changes.
- Update `BLINK.md` when Blink's own build or release posture changes.
- Update `CHANGELOG.md` for notable repo changes.

Active unfinished work belongs in the workspace root `ROADMAP.md`.
